# 1. Purpose

This document defines the `OrderNode` struct — the unit of data that represents one resting order inside the Order Book.

---

# 2. Struct Definition

```go
type OrderNode struct {
    orderID      uuid.UUID
    userID       uuid.UUID
    marketID     string
    side         SideType        // BUY or SELL
    price        decimal.Decimal
    originalQty  decimal.Decimal
    remainingQty decimal.Decimal
    timestamp    time.Time
    element      *list.Element   // pointer to position in PriceLevel.orders
}
```

---

# 3. Fields

---

## orderID

Type: `uuid.UUID` (UUIDv7)

The unique identifier for this order.

Generated by the **Order Service** before the order is persisted — before `ReserveFunds` is called and before the `OrderCreated` event is published.

The Matching Engine receives `orderID` from the `OrderCreated` event payload. It never generates `orderID` itself.

Used as the key in `Side.orderIndex` for O(1) lookup during cancel.

Carried in `TradeExecuted` and `OrderCancelled` events as the correlation ID.

---

## userID

Type: `uuid.UUID` (UUIDv7)

The user who placed the order.

Used in published events (`TradeExecuted` carries `buyer_id` and `seller_id`; `OrderCancelled` carries `user_id`).

The Matching Engine does not validate the user. That responsibility belongs to the API Gateway and Order Service.

---

## marketID

Type: `string`

The trading pair.

Example: `BTC-USDT`

An order always belongs to exactly one market. This field is denormalised here so that events published by the Matching Engine always carry the market ID without requiring a lookup.

---

## side

Type: `SideType` (enum: `BUY` / `SELL`)

Determines which side of the book this order rests on:

- `BUY` → placed in `bids`
- `SELL` → placed in `asks`

---

## price

Type: `decimal.Decimal`

The limit price for this order.

For a BUY order: the maximum price the user is willing to pay.

For a SELL order: the minimum price the user is willing to accept.

Must be a positive value.

Never changes after the order is inserted into the book.

Used as the key to locate the PriceLevel in `Side.priceLevels`.

---

## originalQty

Type: `decimal.Decimal`

The full quantity requested when the order was placed.

Never modified after insertion.

Stored for reference — used in event payloads and for computing fill percentage.

---

## remainingQty

Type: `decimal.Decimal`

The quantity not yet filled.

Starts equal to `originalQty`.

Reduced on every partial fill.

```
After partial fill of amount X:
    remainingQty -= X
```

When `remainingQty` reaches zero, the order is fully filled and must be removed from the book.

This is the only field that changes during the lifetime of a resting order.

---

## timestamp

Type: `time.Time`

The time at which the order was added to the Order Book by the Matching Engine.

This is the arrival time at the ME, not the time the user submitted the order to the API.

**Determines time priority:**

When two orders share the same price, the one with the earlier timestamp executes first (FIFO).

The timestamp is set once at insertion and never changes — even after a partial fill.

Changing the timestamp on partial fill would move the order to the back of the queue, violating Price-Time Priority.

---

## element

Type: `*list.Element`

A back-pointer to this node's position inside `PriceLevel.orders` — a `*list.List` (Go stdlib doubly-linked list).

This is the single most important field for performance.

**Why it exists:**

Cancel requires removing an order from the linked list. Without this pointer, cancel must scan the entire list to find the node: O(n).

With this pointer, cancel calls `list.Remove(node.element)` directly: O(1).

**How it is set:**

When the order is inserted into a PriceLevel:

```go
node.element = priceLevel.orders.PushBack(node)
```

`PushBack` returns a `*list.Element`. That pointer is stored back on the node.

**Pointer graph:**

```
Side.orderIndex[orderID]  -->  *OrderNode
                                    |
                               node.element  -->  *list.Element  (inside PriceLevel.orders)
                                    ^
                         list.Element.Value  (points back to the same *OrderNode)
```

---

# 4. What OrderNode Does NOT Store

OrderNode is a matching-only structure.

It does not store:

| Field | Reason not stored | Where it lives |
|---|---|---|
| Balance / reserved funds | Not needed for matching | Wallet Service |
| Order status (OPEN, FILLED, etc.) | Not needed for matching | Order Service |
| Fee information | Not applied during matching | Settlement Service |
| User email or profile | Not needed for matching | Auth Service |

The Matching Engine contains only what is needed to determine execution priority and produce trade events.

# 5. Lifecycle

```
OrderCreated event received
        |
        v
OrderNode created from event payload
        |
        v
Inserted into PriceLevel.orders (back of queue)
node.element = list.PushBack(node)
Side.orderIndex[orderID] = node
        |
        |
   +----+-------------------------+
   |                              |
   v                              v
Partially matched            Cancel requested
(remainingQty reduced,       Cancel(orderID) called
 stays in book,              node.element used to
 same queue position)        remove from list: O(1)
   |                              |
   v                              v
Fully matched                 Removed from book
list.Remove(node.element)     list.Remove(node.element)
delete orderIndex[orderID]    delete orderIndex[orderID]
```

After removal, the node is no longer referenced by the book and will be garbage-collected by Go's GC.

---

# 6. Queue Priority

Two properties together determine when an order executes:

1. **price** — better price executes first
2. **timestamp** — earlier arrival executes first when prices are equal

The queue at each PriceLevel is FIFO ordered by arrival time.

An order's position in the queue never changes. It is set at insertion and never updated.

A partial fill reduces `remainingQty` in-place. The order is not re-inserted. Its queue position and timestamp remain the same.
