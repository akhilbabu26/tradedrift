# 1. Purpose

This document defines the `OrderNode` struct ‚Äî the unit of data representing one resting order inside the Order Book.

---

# 2. Struct Definition

```go
type OrderNode struct {
    orderID      uuid.UUID
    userID       uuid.UUID
    marketID     string
    side         SideType
    price        decimal.Decimal
    originalQty  decimal.Decimal
    remainingQty decimal.Decimal
    timestamp    time.Time
    element      *list.Element
}
```

## orderID

UUIDv7. Generated by the Order Service before `OrderCreated` is published. The ME receives it from the event payload ‚Äî it never generates `orderID` itself.

Used as the key in `OrderBook.orderIndex` for O(1) cancel. Carried in `TradeExecuted` and `OrderCancelled` events as the correlation ID.

---

## userID

UUIDv7. The user who placed the order. Included in published events. The ME does not validate the user.

---

## marketID

The trading pair (e.g. `BTC-USDT`). Denormalised so published events always carry the market ID without a lookup.

---

## side

`BUY` ‚Üí placed in `bids`. `SELL` ‚Üí placed in `asks`. After the book-level `orderIndex` lookup, `node.side` tells the algorithm which side's `priceLevels` to use.

---

## price

The limit price. For BUY: maximum the user will pay. For SELL: minimum the user will accept. Never changes after insertion. Used as the key to locate the `PriceLevel` in `Side.priceLevels`.

---

## originalQty

Full quantity requested at order creation. Never modified. Stored for event payloads and fill-percentage reference.

---

## remainingQty

Quantity not yet filled. Starts equal to `originalQty`. Reduced on every partial fill. When it reaches zero, the order is fully filled and removed from the book.

---

## timestamp

Time the order was added to the Order Book by the Matching Engine (arrival time at ME, not the user's submission time).

Determines time priority: when two orders share the same price, the earlier timestamp executes first. Set once at insertion ‚Äî never changes, even after partial fills.

---

## element

`*list.Element` ‚Äî a back-pointer to this node's position inside `PriceLevel.orders`.

Without it, cancel must scan the entire linked list to find the node: O(n).
With it, `list.Remove(node.element)` removes the node directly: O(1).

Set at insertion: `node.element = priceLevel.orders.PushBack(node)`

# 6. Queue Priority

Two properties determine execution order:

1. **price** ‚Äî better price executes first across levels
2. **timestamp** ‚Äî earlier arrival executes first within a level (FIFO)

Queue position is set at insertion and **never changes**. Partial fills reduce `remainingQty` in-place ‚Äî the node is never re-inserted. Re-inserting would move the order to the back of the queue, violating Price-Time Priority.
